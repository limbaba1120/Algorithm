# 유연근무제

- 문제 번호: 388351
- 난이도: Level 1
- 풀이 날짜: 2026-07-22
- 문제 유형: 구현, 시간 계산, 요일 계산
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/388351

## 핵심 아이디어

- 출근 시각을 모두 분 단위로 변환한다.
- 출근 희망 시각에 10분을 더해 출근 인정 시각을 구한다.
- 시작 요일과 이벤트가 시작된 후 지난 날짜를 이용해 현재 요일을 계산한다.
- 토요일과 일요일의 출근 기록은 검사하지 않는다.
- 평일에 하루라도 출근 인정 시각보다 늦게 출근했다면 상품을 받을 수 없다.

## 시간 변환

시간은 `시 * 100 + 분` 형태로 주어지므로 다음과 같이 분 단위로 변환한다.

```java
int minutes = time / 100 * 60 + time % 100;
```

예를 들어 `958`은 다음과 같이 변환된다.

```text
9 * 60 + 58 = 598분
```

따라서 출근 희망 시각이 `958`이라면 출근 인정 시각은 `608분`, 즉 `10시 8분`이다.

## 요일 계산

문제에서는 월요일부터 일요일까지를 `1~7`로 표현한다. 이벤트의 `j`번째 기록에 해당하는 요일은 다음과 같이 구한다.

```java
int day = (startday - 1 + j) % 7 + 1;
```

- `startday - 1`: 요일 범위를 `1~7`에서 `0~6`으로 변경한다.
- `+ j`: 시작일로부터 `j`일만큼 이동한다.
- `% 7`: 일요일 다음에 다시 월요일로 돌아가게 한다.
- `+ 1`: 요일 범위를 다시 `1~7`로 변경한다.

## 주의할 점

출근 인정 시각과 정확히 같은 시각에 출근했다면 지각이 아니다. 따라서 늦었는지는 `>`로 비교한다.

```java
if (arrivalTime > deadline) {
    // 지각
}
```

## 복잡도

직원 수를 `n`이라고 할 때 각 직원의 7일 기록을 확인하므로 시간 복잡도는 `O(n)`이고, 추가 공간 복잡도는 `O(1)`이다.
