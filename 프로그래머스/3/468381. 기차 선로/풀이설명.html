기차 선로

빈칸에 선로를 배치하는 백트래킹부터 연결 가지치기, 기차 시뮬레이션까지 Java 코드와 함께 단계별로 이해합니다.

Level 3백트래킹비트마스크시뮬레이션
목차
1. 문제 요약 2. 전체 전략 3. 선로 표현 4. 방향 확인 5. 백트래킹 6. 배치 검사 7. 기차 이동 8. 코드 조립하기 9. 전체 코드 10. 최종 정리

1. 문제를 간단히 요약하면

빈칸에 선로를 적절히 놓아서, 출발한 기차가 목적지까지 모든 선로를 지나도록 만드는 배치의 수를 구하는 문제입니다.

격자의 값

  • 0: 빈칸
  • -1: 장애물
  • 1~7: 이미 놓인 선로

빈칸의 선택

  • 선로를 놓지 않는다.
  • 1~7번 선로 중 하나를 놓는다.

정답이 되기 위한 조건

  1. 이웃한 선로의 연결 방향이 맞아야 합니다.
  2. 기차가 오른쪽 아래 목적지에 도착해야 합니다.
  3. 격자에 놓인 모든 선로를 한 번 이상 지나야 합니다.
  4. 3번 십자 선로는 상하좌우 모두 연결되어야 합니다.
중요한 구분선로끼리 연결되는 것과 기차가 모든 선로를 실제로 지나는 것은 서로 다른 조건입니다.

2. 전체 풀이 전략

1선로 배치

백트래킹으로 빈칸마다 빈칸 또는 1~7번 선로를 시도합니다.

2기차 이동

모든 칸을 정했다면 기차를 직접 움직여 최종 조건을 검사합니다.

선로 선택→연결 검사→다음 칸→기차 시뮬레이션

실제 함수 호출 순서

solution(grid) │ ├─ 입력을 original과 board에 저장 │ └─ dfs(0) │ ├─ 빈칸: canPlace()로 0~7번 선택 검사 │ └─ 가능하면 dfs(index + 1) │ ├─ 장애물·고정 선로: 바로 dfs(index + 1) │ └─ index == n × m └─ canTravelAllTracks() ├─ 기차를 목적지까지 이동 └─ 모든 선로 방문 여부 확인

빈칸 하나에는 8가지 선택이 있지만, 잘못된 연결을 발견하는 즉시 탐색을 중단합니다. 전체 격자가 최대 20칸이므로 이 가지치기가 적용된 백트래킹을 사용할 수 있습니다.

위쪽 선로 │ 현재 선로 ── ← 연결이 끊겼으므로 남은 칸을 볼 필요가 없음

3. 선로를 하나의 숫자로 표현하기

선로 하나에는 위·오른쪽·아래·왼쪽의 연결 정보가 필요합니다. 네 방향을 네 개의 전등 스위치라고 생각해 봅시다.

방향방향 번호비트값2진수
위010001
오른쪽120010
아래240100
왼쪽381000

가로 선로는 오른쪽과 왼쪽 스위치가 켜져 있으므로 0010 | 1000 = 1010입니다. 이렇게 여러 상태를 한 정수에 저장하는 방법을 비트마스크라고 합니다.

선로 비트마스크 
private static final int UP = 1;
private static final int RIGHT = 2;
private static final int DOWN = 4;
private static final int LEFT = 8;

private static final int[] TYPE = {
    0,                              // 빈칸
    RIGHT | LEFT,                   // 1번: 가로
    UP | DOWN,                      // 2번: 세로
    UP | RIGHT | DOWN | LEFT,       // 3번: 십자
    UP | LEFT,                      // 4번
    UP | RIGHT,                     // 5번
    RIGHT | DOWN,                   // 6번
    DOWN | LEFT                     // 7번
};

4. 특정 방향이 열려 있는지 확인하기

hasDirection(mask, direction)은 “이 선로가 해당 방향으로 연결되어 있는가?”만 답하는 함수입니다.

가로 선로 1010 오른쪽 확인값 0010 AND 결과 0010 → 0이 아니므로 오른쪽으로 연결됨
가로 선로 1010 위쪽 확인값 0001 AND 결과 0000 → 0이므로 위쪽으로 연결되지 않음
hasDirection() 
private boolean hasDirection(int mask, int direction) {
    // 1 << direction으로 검사할 방향의 비트를 만든다.
    int directionBit = 1 << direction;

    // 결과가 0이 아니면 그 방향의 스위치가 켜져 있다.
    return (mask & directionBit) != 0;
}
읽는 방법(mask & (1 << direction)) != 0은 “네 방향 스위치 중 확인할 방향의 스위치가 켜져 있는가?”라는 뜻입니다.

5. 백트래킹으로 선로 배치하기

dfs(index)는 index번째 칸부터 가능한 선로를 놓고, 완성할 수 있는 정답 배치 수를 반환합니다.

현재 빈칸 ├─ 빈칸 유지 ├─ 1번 선로 ├─ 2번 선로 ├─ ... └─ 7번 선로
가지치기선로를 하나 선택할 때마다 canPlace()를 호출합니다. 주변과 연결될 수 없는 선택은 다음 칸으로 진행하지 않습니다.
dfs()의 핵심 
private int dfs(int index) {
    // 모든 칸을 정했다면 기차를 실제로 움직인다.
    if (index == n * m) {
        return canTravelAllTracks() ? 1 : 0;
    }

    int r = index / m;
    int c = index % m;

    // 장애물과 고정 선로는 그대로 지나간다.
    if (original[r][c] != 0) {
        return dfs(index + 1);
    }

    int count = 0;

    for (int type = 0; type <= 7; type++) {
        int mask = TYPE[type];

        if (!canPlace(r, c, mask)) {
            continue; // 불가능한 선택 제거
        }

        board[r][c] = mask;
        count += dfs(index + 1);
    }

    board[r][c] = 0; // 되돌리기
    return count;
}
고정 선로의 예외출발점의 1번 선로는 왼쪽 연결이 격자 밖을 향합니다. 따라서 처음부터 주어진 고정 선로에는 새 선로용 경계 검사를 적용하지 않고 그대로 지나갑니다.

6. 현재 선로를 놓을 수 있는지 검사하기

이웃한 두 칸은 서로 마주 보는 방향의 연결 여부가 같아야 합니다.

현재 칸의 오른쪽 O ↔ 오른쪽 칸의 왼쪽 O : 정상 현재 칸의 오른쪽 O ↔ 오른쪽 칸의 왼쪽 X : 실패 현재 칸의 오른쪽 X ↔ 오른쪽 칸의 왼쪽 O : 실패 현재 칸의 오른쪽 X ↔ 오른쪽 칸의 왼쪽 X : 정상

반대 방향은 (d + 2) % 4로 구합니다. 위(0)와 아래(2), 오른쪽(1)과 왼쪽(3)이 서로 반대입니다.

canPlace()의 검사 순서

  1. 현재 선로가 격자 밖을 향하는가?
  2. 현재 선로가 장애물을 향하는가?
  3. 이미 결정된 위쪽·왼쪽 칸과 연결이 맞는가?
  4. 처음부터 주어진 고정 선로와 연결이 맞는가?
canPlace()의 핵심 비교 
boolean currentConnected = hasDirection(mask, d);
int opposite = (d + 2) % 4;
boolean neighborConnected =
    hasDirection(neighborMask, opposite);

// 한쪽만 열려 있으면 연결이 끊긴다.
if (currentConnected != neighborConnected) {
    return false;
}

7. 기차를 직접 움직여 보기

연결 검사만 통과했다고 정답은 아닙니다. 목적지까지 가지 않는 별도 선로가 있을 수 있기 때문입니다.

출발 ───── 목적지 ┌──┐ └──┘ ← 기차가 지나지 않는 별도 선로

기차의 상태

기차의 상태는 (행, 열, 진행 방향)입니다. 같은 십자 칸도 가로로 통과할 때와 세로로 통과할 때가 다르므로 방향까지 방문 상태에 포함합니다.

방향까지 기록하는 방문 배열 
boolean[][] visitedCell = new boolean[n][m];
boolean[][][] visitedState = new boolean[n][m][4];

다음 방향 결정

일반 선로

기차 → ┐ ↓

들어온 면을 제외한 나머지 연결 방향으로 나갑니다.

십자 선로

→ ┼ → ↓ ┼ ↓

현재 진행 방향을 그대로 유지해 직진합니다.

무한 순환 판별같은 칸에 같은 방향으로 다시 도착하면 이후 움직임도 완전히 같으므로 실패입니다.

8. 코드를 처음부터 하나씩 조립하기

이 절만 순서대로 읽어도 전체 코드가 왜 필요한지 이해할 수 있도록, 선언부터 반환까지 빠짐없이 조립합니다.

먼저 전체 순서를 기억하세요상수와 배열 준비 → 입력 변환 → DFS로 선로 선택 → 배치 가능 여부 검사 → 완성된 배치에서 기차 이동 → 성공한 배치 수 반환
1

방향을 정의한다

먼저 프로그램에서 위·오른쪽·아래·왼쪽을 표현할 숫자가 필요합니다.

1단계 코드 
private static final int UP = 1;
private static final int RIGHT = 2;
private static final int DOWN = 4;
private static final int LEFT = 8;

private static final int[] DR = {-1, 0, 1, 0};
private static final int[] DC = {0, 1, 0, -1};
코드를 말로 읽으면: “연결 상태에는 1, 2, 4, 8을 사용하고, 이동 방향 번호는 0=위, 1=오른쪽, 2=아래, 3=왼쪽으로 사용한다.”
  • UP, RIGHT, DOWN, LEFT는 선로가 열린 방향을 저장할 때 사용합니다.
  • DR, DC는 기차나 이웃 좌표를 실제로 한 칸 이동할 때 사용합니다.
  • 방향 번호의 순서를 두 배열에서 똑같이 유지해야 합니다.
2

선로 번호를 연결 방향으로 바꾼다

문제에서는 선로를 1~7번으로 주지만, 프로그램에서는 어느 방향으로 연결되는지가 더 중요합니다.

2단계 코드 
private static final int[] TYPE = {
    0,
    RIGHT | LEFT,              // 1번: 오른쪽 + 왼쪽
    UP | DOWN,                 // 2번: 위 + 아래
    UP | RIGHT | DOWN | LEFT,  // 3번: 모든 방향
    UP | LEFT,
    UP | RIGHT,
    RIGHT | DOWN,
    DOWN | LEFT
};
예시: TYPE[1]은 가로 선로입니다. 값은 RIGHT | LEFT, 즉 오른쪽과 왼쪽으로 연결되었다는 뜻입니다.
TYPE[1] = RIGHT | LEFT = 0010 | 1000 = 1010 TYPE[2] = UP | DOWN = 0001 | 0100 = 0101 TYPE[3] = 네 방향 모두 = 1111

TYPE[0] = 0도 중요합니다. DFS의 type = 0 선택은 “이 빈칸에 아무 선로도 놓지 않는다”는 의미입니다.

3

멤버 변수와 입력 격자를 준비한다

original은 원래 입력을 보관하고, board에는 실제 탐색 중인 비트마스크 선로를 저장합니다.

3단계 코드: solution() 전체 시작 과정 
private int n;
private int m;
private int[][] original;
private int[][] board;

public int solution(int[][] grid) {
    n = grid.length;
    m = grid[0].length;

    original = new int[n][m];
    board = new int[n][m];

    for (int r = 0; r < n; r++) {
        for (int c = 0; c < m; c++) {
            original[r][c] = grid[r][c];

            if (grid[r][c] == -1) {
                board[r][c] = -1; // 장애물
            } else if (grid[r][c] > 0) {
                board[r][c] = TYPE[grid[r][c]]; // 고정 선로
            }
            // 원래 빈칸이면 int 배열 기본값 0이 그대로 남는다.
        }
    }

    return dfs(0); // 0번 칸부터 탐색 시작
}
왜 배열이 두 개일까? original을 보면 원래 빈칸인지 고정 선로인지 알 수 있고, board를 보면 현재 어떤 선로를 선택했는지 알 수 있습니다.
  • original[r][c] == 0: DFS가 선택해야 하는 원래 빈칸입니다.
  • original[r][c] == -1: 바꿀 수 없는 장애물입니다.
  • original[r][c] > 0: 바꿀 수 없는 고정 선로입니다.
  • board의 양수는 선로 번호가 아니라 변환된 방향 비트마스크입니다.
4

DFS가 한 칸씩 이동하게 만든다

아직 선로 선택은 하지 않고, 먼저 격자를 순서대로 방문하는 뼈대를 만듭니다.

4단계 코드 
private int dfs(int index) {
    if (index == n * m) {
        return canTravelAllTracks() ? 1 : 0;
    }

    int r = index / m;
    int c = index % m;

    if (original[r][c] != 0) {
        return dfs(index + 1);
    }

    // 다음 단계에서 빈칸 선택 코드를 넣는다.
}
코드를 말로 읽으면: “현재 번호를 행과 열로 바꾸고, 장애물이나 고정 선로라면 건드리지 말고 다음 칸으로 간다. 모든 칸을 봤다면 기차를 움직인다.”
왜 index == n * m일까?n * m은 전체 칸 수입니다. 칸 번호는 0부터 n*m-1까지이므로, index가 n*m이 되었다면 모든 칸을 처리한 것입니다.
n = 2, m = 3일 때 index 0 1 2 3 4 5 dfs(6)에 도착 → 6 == n × m → 0~5번 칸을 모두 처리함

canTravelAllTracks() ? 1 : 0은 완성된 현재 배치가 정답이면 1개, 아니면 0개라는 뜻입니다.

왜 고정 선로는 canPlace()를 호출하지 않을까?출발점의 1번 선로는 모양상 왼쪽도 열려 있지만, 기차는 오른쪽으로 출발하므로 왼쪽의 격자 밖 연결을 허용해야 합니다. 고정 선로는 그대로 두고 최종 시뮬레이션에서 실제 경로를 검사합니다.
5

빈칸에서 8가지 선택을 시도한다

원래 빈칸인 곳에서는 빈칸 유지와 1~7번 선로를 하나씩 선택합니다.

5단계 코드 
int count = 0;

for (int type = 0; type <= 7; type++) {
    int mask = TYPE[type];

    if (!canPlace(r, c, mask)) {
        continue;
    }

    board[r][c] = mask;          // 선택하기
    count += dfs(index + 1); // 다음 칸
}

board[r][c] = 0;             // 되돌리기
return count;
핵심: 선택 → 다음 칸 탐색 → 되돌리기의 반복입니다. canPlace()가 false라면 그 선택은 더 진행하지 않습니다.

반환값이 더해지는 과정

현재 빈칸에서 type 0 선택의 정답 수 = 2 type 1 선택의 정답 수 = 0 type 2 선택의 정답 수 = 3 나머지 선택의 정답 수 = 0 count = 2 + 0 + 3 + 0 ... = 5

즉, count += dfs(index + 1)은 현재 선택에서 만들어지는 모든 정답 수를 합치는 코드입니다. 함수 끝의 board[r][c] = 0은 부모 DFS로 돌아가기 전에 현재 선택을 지우는 되돌리기입니다.

6

현재 선로와 이웃 선로를 비교한다

현재 선로가 오른쪽으로 열려 있다면 오른쪽 이웃은 왼쪽으로 열려 있어야 합니다.

6단계 코드: canPlace() 전체 흐름 
private boolean canPlace(int r, int c, int mask) {
    for (int d = 0; d < 4; d++) {
        boolean currentConnected = hasDirection(mask, d);
        int nr = r + DR[d];
        int nc = c + DC[d];

        // ① 격자 밖으로 선로가 나가면 실패
        if (!isInside(nr, nc)) {
            if (currentConnected) return false;
            continue;
        }

        // ② 장애물 쪽으로 선로가 나가면 실패
        if (original[nr][nc] == -1) {
            if (currentConnected) return false;
            continue;
        }

        // ③ 위·왼쪽은 이미 처리한 칸
        boolean alreadyDecided =
            nr < r || (nr == r && nc < c);

        // ④ 입력에 있던 고정 선로는 모양을 이미 알고 있음
        boolean fixedTrack = original[nr][nc] > 0;

        if (alreadyDecided || fixedTrack) {
            int neighborMask = fixedTrack
                ? TYPE[original[nr][nc]]
                : board[nr][nc];

            int opposite = (d + 2) % 4;
            boolean neighborConnected =
                hasDirection(neighborMask, opposite);

            // ⑤ 한쪽만 열려 있으면 실패
            if (currentConnected != neighborConnected) {
                return false;
            }
        }
    }

    return true;
}
예시: 현재 선로의 오른쪽이 열려 있다면 오른쪽 칸의 왼쪽도 열려 있어야 합니다. 현재만 열렸거나 이웃만 열렸다면 false를 반환합니다.
왜 오른쪽·아래쪽 빈칸은 바로 비교하지 않을까?아직 어떤 선로가 놓일지 정하지 않았기 때문입니다. 나중에 그 칸을 처리할 때 현재 칸과 연결을 검사합니다.

canPlace()가 true/false를 반환하는 예

예 1) 현재 ─ 장애물 현재 선로가 오른쪽으로 열림 + 오른쪽이 장애물 → false 예 2) 왼쪽 선로 ─ 현재 선로 왼쪽 선로의 오른쪽 O + 현재 선로의 왼쪽 O → 계속 검사 예 3) 왼쪽 선로 ─ 현재 선로 │ 왼쪽 선로의 오른쪽 O + 현재 선로의 왼쪽 X → false 네 방향을 모두 통과 → true

return true는 현재 배치가 최종 정답이라는 뜻이 아닙니다. 지금까지 확인 가능한 주변 연결에 모순이 없다는 뜻일 뿐입니다.

7

기차의 다음 방향을 찾는다

기차가 현재 칸에 들어온 면을 구한 뒤, 일반 선로에서는 다른 연결 방향으로 나갑니다.

7단계 코드: 이동 준비와 반복 검사 
boolean[][] visitedCell = new boolean[n][m];
boolean[][][] visitedState = new boolean[n][m][4];

int r = 0;
int c = 0;
int direction = 1; // 오른쪽으로 출발

while (true) {
    // 격자 밖 또는 선로가 없는 곳이면 실패
    if (!isInside(r, c) || board[r][c] <= 0) {
        return false;
    }

    // 같은 칸에 같은 방향으로 다시 오면 무한 반복
    if (visitedState[r][c][direction]) {
        return false;
    }

    visitedState[r][c][direction] = true;
    visitedCell[r][c] = true;

    // 목적지에 도착하면 이동 반복을 끝냄
    if (r == n - 1 && c == m - 1) {
        break;
    }
방문 배열이 두 개인 이유: visitedCell은 모든 선로를 지났는지 확인하고, visitedState는 같은 위치·방향이 반복되는 무한 순환을 찾습니다.
  • visitedCell[r][c]: 이 칸을 한 번이라도 지났는가?
  • visitedState[r][c][direction]: 이 칸에 같은 진행 방향으로 들어온 적이 있는가?
  • 십자 선로는 같은 칸을 가로와 세로로 각각 지날 수 있으므로 방향까지 기록해야 합니다.
7단계 코드: 다음 방향을 찾아 한 칸 이동 
    int mask = board[r][c];
    int enteredSide = (direction + 2) % 4;
    int nextDirection = -1;

    if (!hasDirection(mask, enteredSide)) {
        return false;
    }

    if (mask == TYPE[3]) {
        nextDirection = direction; // 십자 선로는 직진
    } else {
        for (int d = 0; d < 4; d++) {
            if (d != enteredSide && hasDirection(mask, d)) {
                nextDirection = d;
                break;
            }
        }
    }

    if (nextDirection == -1) {
        return false;
    }

    r += DR[nextDirection];
    c += DC[nextDirection];
    direction = nextDirection;
}
예시: 오른쪽으로 움직여 왔다면 현재 칸의 왼쪽 면으로 들어온 것입니다. 일반 선로에서는 왼쪽을 제외한 다른 연결 방향으로 나가고, 십자 선로에서는 그대로 오른쪽으로 직진합니다.
direction = 오른쪽(1) enteredSide = (1 + 2) % 4 = 왼쪽(3) 현재 선로가 왼쪽 + 아래로 연결됨 → 왼쪽은 들어온 면이므로 제외 → 아래쪽이 nextDirection → r += 1, c += 0으로 아래 칸 이동

nextDirection == -1이면 나갈 연결 방향을 찾지 못한 것이므로 실패합니다. 방향을 찾았다면 DR, DC로 한 칸 이동하고 반복문 처음으로 돌아갑니다.

목적지 도착 후 모든 선로 방문을 확인한다

목적지에 도착했어도 사용하지 않은 선로가 하나라도 있으면 정답이 아닙니다.

8단계 코드 
for (int r = 0; r < n; r++) {
    for (int c = 0; c < m; c++) {
        if (board[r][c] > 0 && !visitedCell[r][c]) {

return true;
최종 성공: 목적지에 도착했고, 놓인 모든 선로의 visitedCell이 true일 때만 한 가지 정답 배치로 셉니다.
목적지 도착만으로는 부족합니다기차가 목적지에 도착해도 다른 위치에 방문하지 않은 선로가 있으면 false입니다. 목적지 도착 후 이중 반복문을 실행하는 이유입니다.

작은 예제로 전체 호출 흐름을 따라간다

아래는 정확한 정답 개수를 계산하려는 예가 아니라, 함수들이 어떤 순서로 호출되는지 보여주는 예입니다.

입력 [1, 0] [0, 2] ① solution() 1번과 2번 고정 선로를 비트마스크로 변환 dfs(0) 호출 ② dfs(0) (0,0)은 고정 선로 → dfs(1) ③ dfs(1) (0,1)은 빈칸 → type 0~7을 차례대로 시도 각 선택마다 canPlace(0,1,mask) 호출 ④ 선택 하나가 가능하면 dfs(2) (1,0)도 빈칸 → 다시 type 0~7 시도 ⑤ dfs(3) (1,1)은 고정 선로 → dfs(4) ⑥ dfs(4) 4 == n × m → 모든 칸 결정 완료 canTravelAllTracks() 호출 ⑦ 시뮬레이션 결과 성공 → 1 반환 실패 → 0 반환 ⑧ 이전 DFS로 돌아가 다음 선로 모양을 계속 시도 모든 반환값을 count에 합산
전체를 한 문장으로: solution()이 준비하고, dfs()가 배치를 만들고, canPlace()가 불가능한 배치를 줄이고, canTravelAllTracks()가 완성된 배치를 최종 판정합니다.
이제 전체 코드를 읽는 순서solution()으로 준비하고 → dfs()로 선로를 놓고 → canPlace()로 가지치기하고 → canTravelAllTracks()로 정답을 확인합니다.

9. 전체 Java 코드

코드는 실행되는 역할별로 나누어 읽으면 쉽습니다.

solution()입력을 탐색에 사용할 형태로 준비하고 DFS를 시작합니다.
dfs()빈칸에 가능한 선로를 놓아 전체 배치를 만듭니다.
canPlace()현재 선로가 주변과 연결될 수 있는지 검사합니다.
canTravelAllTracks()완성된 배치에서 기차를 직접 움직입니다.
도우미 함수방향 연결 여부와 격자 범위를 확인합니다.
Solution.java 
class Solution {
    private static final int UP = 1;
    private static final int RIGHT = 2;
    private static final int DOWN = 4;
    private static final int LEFT = 8;

    // 0=위, 1=오른쪽, 2=아래, 3=왼쪽
    private static final int[] DR = {-1, 0, 1, 0};
    private static final int[] DC = {0, 1, 0, -1};

    private static final int[] TYPE = {
        0,
        RIGHT | LEFT,
        UP | DOWN,
        UP | RIGHT | DOWN | LEFT,
        UP | LEFT,
        UP | RIGHT,
        RIGHT | DOWN,
        DOWN | LEFT
    };

    private int n;
    private int m;
    private int[][] original;
    private int[][] board;

    public int solution(int[][] grid) {
        n = grid.length;
        m = grid[0].length;
        original = new int[n][m];
        board = new int[n][m];

        for (int r = 0; r < n; r++) {
            for (int c = 0; c < m; c++) {
                original[r][c] = grid[r][c];

                if (grid[r][c] == -1) {
                    board[r][c] = -1;
                } else if (grid[r][c] > 0) {
                    board[r][c] = TYPE[grid[r][c]];
                }
            }
        }

        return dfs(0);
    }

    private int dfs(int index) {
        if (index == n * m) {
            return canTravelAllTracks() ? 1 : 0;
        }

        int r = index / m;
        int c = index % m;

        // 장애물과 처음부터 주어진 선로는 그대로 지나간다.
        if (original[r][c] != 0) {
            return dfs(index + 1);
        }

        int count = 0;

        // 빈칸 또는 1~7번 선로를 하나씩 시도한다.
        for (int type = 0; type <= 7; type++) {
            int mask = TYPE[type];

            if (!canPlace(r, c, mask)) {
                continue;
            }

            board[r][c] = mask;
            count += dfs(index + 1);
        }

        board[r][c] = 0;
        return count;
    }

    private boolean canPlace(int r, int c, int mask) {
        for (int d = 0; d < 4; d++) {
            boolean currentConnected = hasDirection(mask, d);
            int nr = r + DR[d];
            int nc = c + DC[d];

            if (!isInside(nr, nc)) {
                if (currentConnected) return false;
                continue;
            }

            if (original[nr][nc] == -1) {
                if (currentConnected) return false;
                continue;
            }

            boolean alreadyDecided = nr < r || (nr == r && nc < c);
            boolean fixedTrack = original[nr][nc] > 0;

            if (alreadyDecided || fixedTrack) {
                int neighborMask = fixedTrack
                    ? TYPE[original[nr][nc]]
                    : board[nr][nc];

                int opposite = (d + 2) % 4;
                boolean neighborConnected =
                    hasDirection(neighborMask, opposite);

                if (currentConnected != neighborConnected) {
                    return false;
                }
            }
        }

        return true;
    }

    private boolean canTravelAllTracks() {
        boolean[][] visitedCell = new boolean[n][m];
        boolean[][][] visitedState = new boolean[n][m][4];

        int r = 0;
        int c = 0;
        int direction = 1; // 오른쪽으로 출발

        while (true) {
            if (!isInside(r, c) || board[r][c] <= 0) {
                return false;
            }

            if (visitedState[r][c][direction]) {
                return false;
            }

            visitedState[r][c][direction] = true;
            visitedCell[r][c] = true;

            if (r == n - 1 && c == m - 1) {
                break;
            }

            int mask = board[r][c];
            int enteredSide = (direction + 2) % 4;

            if (!hasDirection(mask, enteredSide)) {
                return false;
            }

            int nextDirection = -1;

            if (mask == TYPE[3]) {
                // 십자 선로에서는 직진한다.
                nextDirection = direction;
            } else {
                // 들어온 면이 아닌 다른 연결 방향으로 나간다.
                for (int d = 0; d < 4; d++) {
                    if (d != enteredSide && hasDirection(mask, d)) {
                        nextDirection = d;
                        break;
                    }
                }
            }

            if (nextDirection == -1) {
                return false;
            }

            r += DR[nextDirection];
            c += DC[nextDirection];
            direction = nextDirection;
        }

        // 놓인 모든 선로를 지나갔는지 확인한다.
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < m; j++) {
                if (board[i][j] > 0 && !visitedCell[i][j]) {
                    return false;
                }
            }
        }

        return true;
    }

    private boolean hasDirection(int mask, int direction) {
        int directionBit = 1 << direction;
        return (mask & directionBit) != 0;
    }

    private boolean isInside(int r, int c) {
        return 0 <= r && r < n && 0 <= c && c < m;
    }
}

10. 최종 정리

빈칸마다 8가지 선택→연결 불가 즉시 제거→완성 후 기차 이동→정답 개수 증가
  1. 선로는 네 방향의 연결 정보를 가진 비트마스크로 표현합니다.
  2. dfs()가 모든 가능한 선로 배치를 만듭니다.
  3. canPlace()가 잘못된 연결을 조기에 제거합니다.
  4. canTravelAllTracks()가 목적지 도착과 전체 선로 방문을 확인합니다.
